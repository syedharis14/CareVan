# Demo-day runbook — CareVan sales demo at the school gate

The demo is a **real trip** on the demo van through the production pipeline: the van moves along a
real Gulberg route, BOARDED pushes to the parent phone in seconds, REACHED SCHOOL follows ~a minute
later via the real geofence. Total run ≈ 90 s.

## Pre-flight (do once, before you leave)

- [ ] Backend reachable from both the admin laptop and the parent phone (same network / public URL).
- [ ] Database seeded (`pnpm prisma db seed`) → demo entities exist (Demo School, Demo Driver, `DEMO-001`, "Ahmed (Demo)", Demo Parent).
- [ ] Parent phone: CareVan dev-client build installed, signed in as **Demo Parent** `+92 300 0000001` / PIN `1234`.
- [ ] Parent phone: notifications **allowed** (open the app once, accept the prompt) → token registered.
- [ ] Parent phone: lock it, confirm a test push arrives (run the demo once at home end-to-end).
- [ ] Admin laptop: signed into the panel as admin; open **Demo mode** and **Live trips** in two tabs.

## At the gate (the 90-second show)

- [ ] Hand the unlocked-then-locked parent phone to the prospect (or hold it facing them).
- [ ] Admin → **Demo mode** → **▶ Start demo** → chip shows "Running" → "✓ BOARDED fired".
- [ ] Within ~10 s: parent phone buzzes — **"Ahmed (Demo) boarded the van"**. (This is the money moment.)
- [ ] Switch the prospect's attention to the parent app child card: status is **On the van → school** (amber), map shows the van moving along the route.
- [ ] ~60 s after boarding: parent phone buzzes — **"Ahmed (Demo) reached school"**; card flips to **At school** (green). Demo panel shows "✓ REACHED SCHOOL fired".
- [ ] Optional: show the prospect **Live trips** on the laptop — the same van moving in real time.

## Talk track hooks

- Boarded push = "you'll know the instant your child is on the van, every morning."
- Green 'At school' = "and the instant they're safely there — you never have to wonder or call."
- Live map = "and any time in between, you can see exactly where the van is."

## If something goes wrong

- **No BOARDED push in ~15 s:** the app reconciles on open — open the parent app; the card will show the correct status even if the push was missed. (Check the phone had notifications enabled + network.)
- **"Demo data missing" error:** the seed wasn't run on this backend → run `pnpm prisma db seed`.
- **"A demo is already running" (409):** a previous run is still going (~90 s) — wait for it to finish, then start again.
- **Map blank on the phone:** Android needs the Google Maps API key in the build; the status card + pushes still work without it. (Admin Live-trips map uses OpenStreetMap and always works.)

## What the demo proves (and does not touch)

- Runs the **real** trip → ping → geofence → AlertLog → push pipeline; REACHED SCHOOL is derived by
  the same geofence code as production, not a fake timer.
- Demo trips are flagged `isDemo` and **excluded from driver payouts** — running demos never inflates
  anyone's active-days. Demo entities are separate accounts, so no real family sees demo data.
